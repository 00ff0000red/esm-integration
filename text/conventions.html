Conventions¶

The textual format for WebAssembly modules is a rendering of their abstract syntax into S-expressions.

Like the binary format, the text format is defined by an attribute grammar. A text string is a well-formed description of a module if and only if it is generated by the grammar.

Each production of this grammar has exactly one synthesized attribute: the abstract syntax that the respective byte sequence encodes. Thus, the attribute grammar implicitly defines a parsing function.

Except for a few exceptions, the core of the text grammar closely mirrors the grammar of the abstract syntax. However, it also defines a number of forms that are syntactic sugar over the core syntax.

The recommended extension for source files containing WebAssembly modules in text format is “\(\mathtt{.wat}\)”.

Grammar¶

The following conventions are adopted in defining grammar rules for the text format. They mirror the conventions used for abstract syntax and for the binary format. In order to distinguish symbols of the textual syntax from symbols of the abstract syntax, typewriter font is adopted for the former.

  • Terminal symbols are strings of characters enclosed in quotes: \(\def\mathdef1576#1{\mbox{‘}\mathtt{#1}\mbox{’}}\mathdef1576{module}\).
  • Nonterminal symbols are written in typewriter font: \(\mathtt{valtype}, \mathtt{instr}\).
  • Arbitrary white space is allowed in any place where the grammar contains spaces, except where noted otherwise, such as the lexical syntax.
  • \(T^n\) is a sequence of \(n\geq 0\) iterations of \(T\).
  • \(T^\ast\) is a possibly empty sequence of iterations of \(T\). (This is a shorthand for \(T^n\) used where \(n\) is not relevant.)
  • \(T^?\) is an optional occurrence of \(T\). (This is a shorthand for \(T^n\) where \(n \leq 1\).)
  • \(x{:}T\) denotes the same language as the nonterminal \(T\), but also binds the variable \(x\) to the attribute synthesized for \(T\).
  • Productions are written \(\mathtt{name} ::= T \Rightarrow A\), where \(A\) is the attribute that is synthesized for \(\mathtt{name}\), usually from attribute variables bound in \(T\).
  • Some productions are augmented by side conditions in parentheses, which restrict the applicability of the production. They provide a shorthand for a combinatorial expansion of the production into many separate cases.

Note

For example, the textual grammar for value types is given as follows:

\[\begin{split}\begin{array}{llcll@{\qquad\qquad}l} \def\mathdef1528#1{{}}\mathdef1528{value types} & \href{../text/types.html#text-valtype}{\mathtt{valtype}} &::=& \def\mathdef1577#1{\mbox{‘}\mathtt{#1}\mbox{’}}\mathdef1577{i32} &\Rightarrow& \href{../syntax/types.html#syntax-valtype}{\mathsf{i32}} \\ &&|& \def\mathdef1578#1{\mbox{‘}\mathtt{#1}\mbox{’}}\mathdef1578{i64} &\Rightarrow& \href{../syntax/types.html#syntax-valtype}{\mathsf{i64}} \\ &&|& \def\mathdef1579#1{\mbox{‘}\mathtt{#1}\mbox{’}}\mathdef1579{f32} &\Rightarrow& \href{../syntax/types.html#syntax-valtype}{\mathsf{f32}} \\ &&|& \def\mathdef1580#1{\mbox{‘}\mathtt{#1}\mbox{’}}\mathdef1580{f64} &\Rightarrow& \href{../syntax/types.html#syntax-valtype}{\mathsf{f64}} \\ \end{array}\end{split}\]

The textual grammar for limits is defined as follows:

\[\begin{split}\begin{array}{llclll} \def\mathdef1528#1{{}}\mathdef1528{limits} & \href{../text/types.html#text-limits}{\mathtt{limits}} &::=& n{:}\href{../text/values.html#text-int}{\def\mathdef1565#1{{\mathtt{u}#1}}\mathdef1565{\mathtt{32}}} &\Rightarrow& \{ \mathsf{min}~n, \mathsf{max}~\epsilon \} \\ &&|& n{:}\href{../text/values.html#text-int}{\def\mathdef1565#1{{\mathtt{u}#1}}\mathdef1565{\mathtt{32}}}~~m{:}\href{../text/values.html#text-int}{\def\mathdef1565#1{{\mathtt{u}#1}}\mathdef1565{\mathtt{32}}} &\Rightarrow& \{ \mathsf{min}~n, \mathsf{max}~m \} \\ \end{array}\end{split}\]

The variables \(n\) and \(m\) name the attributes of the respective \(\href{../text/values.html#text-int}{\def\mathdef1565#1{{\mathtt{u}#1}}\mathdef1565{\mathtt{32}}}\) nonterminals, which in this case are the actual unsigned integers they decode into. The attribute of the complete production then is the abstract syntax for the limit, expressed in terms of the former values.

Auxiliary Notation¶

When dealing with binary encodings the following notation is also used:

  • \(\epsilon\) denotes the empty byte sequence.

Contexts¶

The text format allows to use symbolic identifiers in place of indices. To resolve these identifiers into concrete indices, some grammar production are indexed by an identifier context that records the declared identifiers in each index space.

It is convenient to define identifier contexts as records \(I\) with abstract syntax:

\[\begin{split}\begin{array}{llll} \def\mathdef1528#1{{}}\mathdef1528{(identifier context)} & I &::=& \begin{array}[t]{l@{~}ll} \{ & \mathsf{types} & (\href{../text/values.html#text-id}{\mathtt{id}}^?)^\ast, \\ & \mathsf{funcs} & (\href{../text/values.html#text-id}{\mathtt{id}}^?)^\ast, \\ & \mathsf{tables} & (\href{../text/values.html#text-id}{\mathtt{id}}^?)^\ast, \\ & \mathsf{mems} & (\href{../text/values.html#text-id}{\mathtt{id}}^?)^\ast, \\ & \mathsf{globals} & (\href{../text/values.html#text-id}{\mathtt{id}}^?)^\ast, \\ & \mathsf{locals} & (\href{../text/values.html#text-id}{\mathtt{id}}^?)^\ast, \\ & \mathsf{labels} & (\href{../text/values.html#text-id}{\mathtt{id}}^?)^\ast ~\} \\ \end{array} \end{array}\end{split}\]

For each index space, an identifier context contains the list of identifiers assigned to the defined indices. Unnamed indices are associated with empty (\(\epsilon\)) entries in these lists.

It is an invariant of the specification that no component contains duplicate identifiers.
