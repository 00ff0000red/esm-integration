Binary Format¶

The binary format for WebAssembly modules is defined as an encoding function that maps abstract syntax to sequences of raw bytes. Some phrases may have several possible encodings, in particular, numbers may be encoded with optional leading zeros. Consequently, the encoding function defined here in fact yields a (non-empty) set of possible byte sequences for each input phrase. Implementations of encoders producing WebAssembly binaries can pick any encoding contained in these sets.

Every byte sequence that is a valid encoding for a given syntactic class uniquely determines a phrase of that syntactic class. Consequently, encoding is invertible in that sense, and decoding is defined implicitly as the respective inverse function. A decoder in an implementation of WebAssembly must accept all possible encodings for a phrase. [1]

[1]Additional encoding layers – for example, introducing compression – may be defined on top the basic representation defined here. However, such layers are outside the scope of the current specification.

Notation¶

The following notation is adopted in defining binary encoding functions.

  • A hexadecimal constant like \(\mathrm{08}\) or \(\mathrm{FF}\) denotes the respective byte value as a singleton set.

  • \(\mathrm{byte}(n)\) denotes the byte value of the natural number \(n\) (for \(0 \leq n < 256\)).

  • \(\def\sem#1{[\![#1]\!]}\sem{x}_{t}\) denotes the set of encodings of the phrase \(x\) of type \(t\). Usually \(t\) is clear from context and will be omitted to avoid clutter.

  • The sequence \(\def\sem#1{[\![#1]\!]}\sem{x}_{t_1}~\def\sem#1{[\![#1]\!]}\sem{y}_{t_2}\) denotes the set of encodings that are the concatenation of an element from \(\def\sem#1{[\![#1]\!]}\sem{x}_{t_1}\) with an element from \(\def\sem#1{[\![#1]\!]}\sem{y}_{t_2}\).

    Note

    For example, the set \(\def\sem#1{[\![#1]\!]}\sem{10}\) of encodings of the \({\mathit{uint}_8}\) value \(10\) is \(\{\mathrm{0A}, \mathrm{8A~00}\}\). The concatenation \(\def\sem#1{[\![#1]\!]}\sem{10}~\def\sem#1{[\![#1]\!]}\sem{10}\) yields the set \(\{\mathrm{0A~0A}, \mathrm{0A~8A~00}, \mathrm{8A~00~0A}, \mathrm{8A~00~8A~00}\}\) of possible encodings.

  • The definition of the encoding function is given in clause form:

    \[\begin{split}\begin{array}{lll} \def\sem#1{[\![#1]\!]}\sem{x}_t &=& \mathit{byte~sequence~encoding~x} \\ \def\sem#1{[\![#1]\!]}\sem{y}_t &=& \mathit{byte~sequence~encoding~y} \\ \end{array}\end{split}\]

    This function may be recursive. When multiple clauses overlap, any clause may be chosen. The result of the encoding function then is the union of all applicable clauses.

    Note

    For example, the \(\mathsf{else}\) opcode may be omitted in the encoding of the \(\mathsf{if}\) instruction if the following instruction sequence is empty:

    \[\begin{split}\begin{array}{lll} \def\sem#1{[\![#1]\!]}\sem{\mathsf{if}~e_1^\ast~\mathsf{else}~e_2^\ast~\mathsf{end}} &=& \dots \\ \def\sem#1{[\![#1]\!]}\sem{\mathsf{if}~e_1^\ast~\mathsf{else}~\mathsf{end}} &=& \dots \\ \end{array}\end{split}\]

Numbers¶

All integral numbers are encoded using the LEB128 variable-length integer encoding, in either unsigned or signed variant.

  • \({\mathit{uint}}_N\): unsigned integers are encoded in unsigned LEB128 format:

    \[\begin{split}\begin{array}{lll@{\qquad\qquad}l} \def\sem#1{[\![#1]\!]}\sem{n}_{{\mathit{uint}}_N} &=& \mathrm{byte}(n) & (n < 128) \\ \def\sem#1{[\![#1]\!]}\sem{m \cdot 128 + n}_{{\mathit{uint}}_N} &=& \mathrm{byte}(n+128)~\def\sem#1{[\![#1]\!]}\sem{m}_{{\mathit{uint}}_N} & (n < 128) \\ \end{array}\end{split}\]

    In the case of a value less than 128, both rules apply, allowing for optional padding \(\mathrm{80}\) bytes.

    As an additional constraint, the total number of bytes encoding a value of type \({\mathit{uint}}_N\) must not exceed \(\mathrm{ceil}(N/7)\) bytes.

  • \({\mathit{sint}}_N\): signed integers are encoded in signed LEB128 format, which uses a 2’s complement representation:

    \[\begin{split}\begin{array}{lll@{\qquad\qquad}l} \def\sem#1{[\![#1]\!]}\sem{n}_{{\mathit{sint}}_N} &=& \mathrm{byte}(n) & (n < 64) \\ \def\sem#1{[\![#1]\!]}\sem{-n}_{{\mathit{sint}}_N} &=& \mathrm{byte}(128-n) & (0 < n \leq 64) \\ \def\sem#1{[\![#1]\!]}\sem{\pm m \cdot 128 + n}_{{\mathit{sint}}_N} &=& \mathrm{byte}(n+128)~\def\sem#1{[\![#1]\!]}\sem{\pm m}_{{\mathit{sint}}_N} & (n < 128) \\ \end{array}\end{split}\]

    In the case of a non-negative value less than 64, both the first and second rule apply, allowing for optional padding \(\mathrm{80}\) bytes. In the case of a negative value greater than or equal to -64, both the second and third rule apply, allowing for optional padding \(\mathrm{FF}\) bytes.

    As an additional constraint, the total number of bytes encoding a value of type \({\mathit{sint}}_N\) must not exceed \(\mathrm{ceil}(N/7)\) bytes.

  • \(\float_N\): floating point values are encoded as a vector of bytes containing their IEEE bit pattern in little endian byte order:

    \[\begin{split}\begin{array}{lll@{\qquad\qquad}l} \def\sem#1{[\![#1]\!]}\sem{z}_{\float_N} &=& \def\sem#1{[\![#1]\!]}\sem{\mathrm{bytes}(z)}_{\mathrm{byte}^\ast} \\ \end{array}\end{split}\]

Sequences and Vectors¶

All vectors are encoded by a their length, succeeded by their list of elements.

\[\begin{split}\begin{array}{lll@{\qquad\qquad}l} \def\sem#1{[\![#1]\!]}\sem{x^n}_{{\mathit{uint}}_N} &=& \def\sem#1{[\![#1]\!]}\sem{x} \\ \end{array}\end{split}\]

  • \(\string\): strings are encoded as a vector of bytes:

    \[\begin{split}\begin{array}{lll@{\qquad\qquad}l} \def\sem#1{[\![#1]\!]}\sem{s} &=& \mathrm{bytes}(z) \\ \end{array}\end{split}\]
