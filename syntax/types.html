Types¶

Value Types¶

Value types classify the individual values that WebAssembly code can compute with and the values that a variable accepts.

\[\begin{split}\begin{array}{llll} \def\vo_id#1{}\vo_id{value types} & \href{../syntax/types.html#syntax-valtype}{\mathit{valtype}} &::=& \href{../syntax/types.html#syntax-valtype}{\mathsf{i32}} ~|~ \href{../syntax/types.html#syntax-valtype}{\mathsf{i64}} ~|~ \href{../syntax/types.html#syntax-valtype}{\mathsf{f32}} ~|~ \href{../syntax/types.html#syntax-valtype}{\mathsf{f64}} \\ \end{array}\end{split}\]

The types \(\href{../syntax/values.html#syntax-int}{\mathit{int}_{32}}\) and \(\href{../syntax/values.html#syntax-int}{\mathit{int}_{64}}\) classify 32 and 64 bit integers, respectively. Integers are not inherently signed or unsigned, their interpretation is determined by individual operations.

The types \(\href{../syntax/values.html#syntax-float}{\mathit{float}_{32}}\) and \(\href{../syntax/values.html#syntax-float}{\mathit{float}_{64}}\) classify 32 and 64 bit floating points, respectively. They correspond to single and double precision floating point types as defined by the IEEE-754 standard

Conventions¶

  • The meta variable \(t\) ranges over value types where clear from context.
  • The notation \(|t|\) denotes the width of a value type in bytes. (That is, \(|\href{../syntax/types.html#syntax-valtype}{\mathsf{i32}}| = |\href{../syntax/types.html#syntax-valtype}{\mathsf{f32}}| = 4\) and \(|\href{../syntax/types.html#syntax-valtype}{\mathsf{i64}}| = |\href{../syntax/types.html#syntax-valtype}{\mathsf{f64}}| = 8\).)

Result Types¶

Result types classify the results of functions or blocks, which is a sequence of values.

\[\begin{split}\begin{array}{llll} \def\vo_id#1{}\vo_id{result types} & \href{../syntax/types.html#syntax-resulttype}{\mathit{resulttype}} &::=& [\href{../syntax/types.html#syntax-valtype}{\mathit{valtype}}^?] \\ \end{array}\end{split}\]

Note

In the current version of WebAssembly, at most one value is allowed as a result. However, this may be generalized to sequences of values in future versions.

Function Types¶

Function types classify the signature of functions, mapping a vector of parameters to a vector of results.

\[\begin{split}\begin{array}{llll} \def\vo_id#1{}\vo_id{function types} & \href{../syntax/types.html#syntax-functype}{\mathit{functype}} &::=& [\href{../syntax/values.html#syntax-vec}{\mathit{vec}}(\href{../syntax/types.html#syntax-valtype}{\mathit{valtype}})] \to [\href{../syntax/values.html#syntax-vec}{\mathit{vec}}(\href{../syntax/types.html#syntax-valtype}{\mathit{valtype}})] \\ \end{array}\end{split}\]

Note

In the current version of WebAssembly, the length of the result type vector of a function may be at most \(1\). This restriction may be removed in future versions.

Memory Types¶

Memory types classify linear memories and their size range.

\[\begin{split}\begin{array}{llll} \def\vo_id#1{}\vo_id{memory types} & \href{../syntax/types.html#syntax-memtype}{\mathit{memtype}} &::=& \href{../syntax/types.html#syntax-limits}{\mathit{limits}} \\ \def\vo_id#1{}\vo_id{limits} & \href{../syntax/types.html#syntax-limits}{\mathit{limits}} &::=& \{ \mathsf{min}~\href{../syntax/values.html#syntax-int}{\mathit{uint}_{32}}, \mathsf{max}~\href{../syntax/values.html#syntax-int}{\mathit{uint}_{32}}^? \} \\ \end{array}\end{split}\]

The limits constrain the minimum and optionally the maximum size of a table. If no maximum is given, the table can grow to any size. Both values are given in units of page size.

Table Types¶

Table types classify tables over elements of element types within a given size range.

\[\begin{split}\begin{array}{llll} \def\vo_id#1{}\vo_id{table types} & \href{../syntax/types.html#syntax-tabletype}{\mathit{tabletype}} &::=& \href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}} \\ \def\vo_id#1{}\vo_id{element types} & \href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}} &::=& \href{../syntax/types.html#syntax-elemtype}{\mathsf{anyfunc}} \\ \end{array}\end{split}\]

Like memories, tables are constrained by limits for their minimum and optionally the maximum size. These sizes are given in numbers of entries.

The element type \(\href{../syntax/types.html#syntax-elemtype}{\mathsf{anyfunc}}\) is the infinite union of all function types. A table of that type thus contains references to functions of heterogeneous type.

Note

In future versions of WebAssembly, additional element types may be introduced.

Global Types¶

Global types classify global variables, which hold a value and can either be mutable or immutable.

\[\begin{split}\begin{array}{llll} \def\vo_id#1{}\vo_id{global types} & \href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}} &::=& \href{../syntax/types.html#syntax-mut}{\mathit{mut}}^?~\href{../syntax/types.html#syntax-valtype}{\mathit{valtype}} \\ \def\vo_id#1{}\vo_id{mutability} & \href{../syntax/types.html#syntax-mut}{\mathit{mut}} &::=& \mathsf{const} ~|~ \href{../syntax/types.html#syntax-mut}{\mathsf{mut}} \\ \end{array}\end{split}\]

External Types¶

External types classify imports and exports and their respective types.

\[\begin{split}\begin{array}{llll} \def\vo_id#1{}\vo_id{external types} & \href{../syntax/types.html#syntax-externtype}{\mathit{externtype}} &::=& \mathsf{func}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}} ~|~ \\&&& \mathsf{table}~\href{../syntax/types.html#syntax-tabletype}{\mathit{tabletype}} ~|~ \\&&& \mathsf{mem}~\href{../syntax/types.html#syntax-memtype}{\mathit{memtype}} ~|~ \\&&& \mathsf{global}~\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}} \\ \end{array}\end{split}\]

Conventions¶

The following auxiliary notation is defined for sequences of external types, filtering out entries of a specific kind in an order-preserving fashion:

  • \(\href{../syntax/types.html#syntax-externtype}{\mathrm{funcs}}(\href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}^\ast) = [\href{../syntax/types.html#syntax-functype}{\mathit{functype}} ~|~ \mathsf{func}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}} \in \href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}^\ast]\)
  • \(\href{../syntax/types.html#syntax-externtype}{\mathrm{tables}}(\href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}^\ast) = [\href{../syntax/types.html#syntax-tabletype}{\mathit{tabletype}} ~|~ \mathsf{table}~\href{../syntax/types.html#syntax-tabletype}{\mathit{tabletype}} \in \href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}^\ast]\)
  • \(\href{../syntax/types.html#syntax-externtype}{\mathrm{mems}}(\href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}^\ast) = [\href{../syntax/types.html#syntax-memtype}{\mathit{memtype}} ~|~ \mathsf{mem}~\href{../syntax/types.html#syntax-memtype}{\mathit{memtype}} \in \href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}^\ast]\)
  • \(\href{../syntax/types.html#syntax-externtype}{\mathrm{globals}}(\href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}^\ast) = [\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}} ~|~ \mathsf{global}~\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}} \in \href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}^\ast]\)
